# Darkness Kingdoms — UI Style and Layout Guide

## Project

**Project:** Darkness Kingdoms

**Target:** Mobile portrait Phaser 3 idle RPG / hero collector / gacha

**Style:** Premium gothic dark fantasy UI

**Mood:** royal, cursed, ancient, luxurious, dangerous, mystical

**References:** AFK Arena-style idle RPG structure, Raid-style premium UI polish, Diablo Immortal-like dark fantasy menu depth

## Core UI Rule

The game uses a persistent mobile RPG shell.

The bottom navigation is the main mode selector.

The center area must always show content that belongs to the currently selected bottom-nav tab.

Do not create a disconnected hub where the center area shows unrelated decorative scenery while the bottom nav opens separate unrelated screens.

## Global Screen Structure

```text
TOP BAR
- ruler profile
- player level / power
- currencies
- settings

CENTER AREA
- changes based on selected bottom-nav tab
- must match the active tab

SIDE SHORTCUTS
- optional secondary shortcuts
- context-aware where possible
- events, mail, quests, offers, rankings, arena, codex, etc.

BOTTOM NAVIGATION
- always visible on major shell screens
- controls the main center content
```

## Locked Bottom Navigation

Use this main bottom navigation structure:

```text
Campaign | Heroes | Summon | Guild | More
```

## Center Content by Bottom Nav

### Campaign

Center area shows:

- campaign chapter map
- chapter background
- stage nodes
- boss node
- reward preview
- formation/start controls

This should eventually replace plain stage lists.

### Heroes

Center area shows:

- hero roster
- selected hero preview
- upgrade entry points
- gear entry points
- kingdom/faction/class/rarity filters

### Summon

Center area shows:

- abyssal summon portal / altar
- active banner
- x1 and x10 summon buttons
- pity/rates/wishlist access
- summon currency display

### Guild

Center area shows:

- guild hall / war council
- guild boss
- guild shop
- guild quests
- guild status

### More

Center area shows secondary systems:

- Arena
- Towers
- World Boss
- Codex / daily tasks
- Elder Tree equivalent
- Settings
- other future systems

## Main Hub Rule

Do not treat Main Hub as a separate disconnected city screen.

If a MainHub scene exists in code, it should behave as the global shell controller or default layout container.

The game should open on an active bottom-nav tab, most likely Campaign, unless a true Home tab is added later with a clear gameplay purpose.

## Safe Area and Measurements

Mobile portrait target:

- Base canvas: 480 x 854
- Respect notches / rounded corners
- Outer padding minimum: 16px
- Top bar target height: about 72px
- Bottom nav target height: about 92px
- Center area uses remaining safe space

## Top Bar

Top bar should contain:

- ruler avatar / profile button
- player name or ruler title
- combat power
- currencies
- small plus buttons where needed
- settings button

Currency examples:

- Gold
- Abyss Crystals
- Summon Seals
- Energy or stamina if used

## Side Shortcuts

Side shortcuts are secondary, not the main navigation.

Possible left side shortcuts:

- Friends
- Arena
- Rankings
- Chat

Possible right side shortcuts:

- Quests
- Mail
- Events
- Battle Pass
- Limited Offers

Rule: avoid visual clutter. Do not show too many badges at once.

## Notification Rules

Use badges only when useful.

Badge meanings:

- Red dot: unclaimed reward
- Gold glow: upgrade available
- Blue pulse: new content
- Countdown: limited-time event

Never overload the screen with notification badges.

## Visual Identity

The UI should feel like:

- premium gothic dark fantasy
- royal dark kingdom
- cursed throne
- abyssal magic
- ancient war
- high-value mobile RPG

Core colors:

- black depth
- royal purple magic
- antique gold trim
- crimson danger/event accents
- muted bone/ivory text

Materials:

- dark carved stone
- antique gold metal
- abyss crystal
- dark enchanted glass
- worn royal banners

## Asset Placeholder Rule

For now, keep UI visuals as placeholders.

Do not overwork Phaser rectangles, gradients, fake borders, or procedural decoration trying to make the game beautiful in code.

Phaser should focus on:

- layout
- hit zones
- state
- readable text
- navigation
- placeholder panels
- functional screen structure

Final beauty should come from proper high-resolution UI assets later.

## Animation Priorities

Only after layout is functional:

- button press scale
- nav selected glow
- reward burst
- currency count-up
- summon portal pulse
- subtle particles
- banner sway
- hero idle breathing later

## UX Rules

- one tap to main systems
- no deep submenu chains early
- keep center content readable
- avoid cluttering the screen with too many shortcuts
- keep top and bottom shell consistent
- selected bottom nav tab must be obvious

## Implementation Rule

When coding UI:

1. Make the layout correct first.
2. Keep all controls functional.
3. Use simple placeholders.
4. Do not polish with complex Phaser shapes.
5. Do not randomly layer PNGs over old UI.
6. Replace old UI intentionally, one screen/tab at a time.
7. After each change, verify with the SPCK browser console.

## Current Priority

First priority is not final art.

First priority is:

```text
Everything in the right place and functional.
```

Build the persistent shell behavior first:

```text
Campaign | Heroes | Summon | Guild | More
```

Then improve each tab one by one.
